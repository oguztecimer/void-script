  <action>
Per user decision in CONTEXT.md: "Add a verification step to confirm wry custom protocol serves `.woff2` files with correct MIME type."

The existing `embedded_assets.rs` uses `mime_guess::from_path(path)` to determine MIME types. The `mime_guess` crate v2 should resolve `.woff2` to `font/woff2`. This task confirms that assumption with a test rather than relying on it silently.

1. **Inspect `crates/voidscript-editor/src/embedded_assets.rs`** -- confirm the `get_asset` function uses `mime_guess::from_path()` to resolve MIME types. It currently does (line 15). No code change needed if `mime_guess` already handles `.woff2`.

2. **Add a Rust unit test** to `embedded_assets.rs` that verifies `.woff2` MIME resolution:
   ```rust
   #[cfg(test)]
   mod tests {
       use super::*;

       #[test]
       fn woff2_mime_type_is_correct() {
           // Verify mime_guess resolves .woff2 to font/woff2.
           // This is critical: if mime_guess ever drops .woff2 support or
           // returns application/octet-stream, fonts will silently fail
           // in the wry WebView.
           let mime = mime_guess::from_path("font.woff2")
               .first_or_octet_stream()
               .to_string();
           assert_eq!(mime, "font/woff2", "mime_guess must resolve .woff2 to font/woff2");
       }

       #[test]
       fn common_web_mime_types_are_correct() {
           // Sanity check other critical asset types served by the custom protocol
           let cases = vec![
               ("index.html", "text/html"),
               ("style.css", "text/css"),
               ("app.js", "application/javascript"),
               ("font.woff", "font/woff"),
           ];
           for (path, expected) in cases {
               let mime = mime_guess::from_path(path)
                   .first_or_octet_stream()
                   .to_string();
               assert_eq!(mime, expected, "MIME mismatch for {path}");
           }
       }
   }
   ```

3. **Run the test** to confirm it passes: `cargo test -p voidscript-editor woff2_mime_type_is_correct`

If the test FAILS (mime_guess does not return `font/woff2`), add an explicit fallback in `get_asset()` before the `mime_guess` call:
```rust
// Explicit MIME overrides for types mime_guess may not handle
let mime = match path.rsplit('.').next() {
    Some("woff2") => "font/woff2".to_string(),
    Some("woff") => "font/woff".to_string(),
    _ => mime_guess::from_path(path)
        .first_or_octet_stream()
        .to_string(),
};
```

Only apply this fallback if the test fails. The test itself must be added regardless.
