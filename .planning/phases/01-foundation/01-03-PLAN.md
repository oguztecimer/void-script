<objective>
Fix traffic light window controls (minimize/maximize/close) that crash the app.

Purpose: UAT revealed that clicking minimize or activating another window crashes the editor, and maximize does not toggle correctly. The root cause is that `handle_window_controls` bypasses Bevy's window management by calling raw winit `set_minimized()`/`set_maximized()` directly on the winit window handle. Bevy 0.15 provides `Window::set_minimized()` and `Window::set_maximized()` on the Bevy `Window` component, which internally sets request flags that Bevy's own winit integration syncs safely on the next frame. The fix is to operate on the Bevy `Window` component instead of the raw winit window.

Output: A working `handle_window_controls` system that uses the Bevy Window component API for minimize/maximize and `AppExit` for close.
</objective>

**Step 2:** Replace the `handle_window_controls` function (lines 124-151) with:

```rust
pub fn handle_window_controls(
    mut events: EventReader<WindowControlEvent>,
    mut editor_windows: Query<(Entity, &mut Window), With<EditorWindow>>,
    mut webview_manager: NonSendMut<WebViewManager>,
    mut commands: Commands,
    mut app_exit_events: EventWriter<AppExit>,
    mut maximized_state: ResMut<MaximizedState>,
) {
    for event in events.read() {
        for (entity, mut window) in editor_windows.iter_mut() {
            match event {
                WindowControlEvent::Minimize => {
                    window.set_minimized(true);
                }
                WindowControlEvent::Maximize => {
                    maximized_state.maximized = !maximized_state.maximized;
                    window.set_maximized(maximized_state.maximized);
                }
                WindowControlEvent::Close => {
                    webview_manager.webviews.remove(&entity);
                    commands.entity(entity).despawn();
                    app_exit_events.send(AppExit::Success);
                }
            }
        }
    }
}
```

Key changes from the original:
1. **Removed** `winit_windows: NonSend&lt;WinitWindows&gt;` parameter -- no longer needed in this function
2. **Changed** `editor_windows` query from `Query&lt;Entity, With&lt;EditorWindow&gt;&gt;` to `Query&lt;(Entity, &amp;mut Window), With&lt;EditorWindow&gt;&gt;` to get mutable access to the Bevy Window component
3. **Added** `mut app_exit_events: EventWriter&lt;AppExit&gt;` -- sends `AppExit::Success` on close for clean shutdown
4. **Added** `mut maximized_state: ResMut&lt;MaximizedState&gt;` -- tracks maximize toggle state
5. `window.set_minimized(true)` calls Bevy's Window method which sets `internal.minimize_request = Some(true)` -- Bevy's winit plugin reads this on next frame and calls the winit API safely
6. `window.set_maximized(...)` calls Bevy's Window method for the same safe path
7. Close still removes the webview and despawns entity, but also sends AppExit for clean shutdown
