**ToolBtn** — A composable icon button that replaces three existing patterns: Header's `ToolBtn` (28x28), Header's `ActionBtn` (28x28 with colored bg), and ScriptList/DebugPanel's `ToolWindowBtn` (small close icon). Per user decision: size prop with default 36px and small 26px for toolbar widgets. Icon-only is the common case.

Props interface:
```typescript
interface ToolBtnProps {
  title: string;
  onClick?: () => void;
  disabled?: boolean;
  active?: boolean;
  size?: 'default' | 'small';    // default=36px, small=26px
  variant?: 'ghost' | 'filled';  // ghost=transparent bg, filled=colored bg
  bgColor?: string;              // for filled variant (e.g., var(--bg-btn-run))
  hoverBgColor?: string;         // for filled variant hover
  iconColor?: string;            // override icon color
  className?: string;            // allow parent CSS class additions (e.g., titlebar-no-drag)
  children: React.ReactNode;
}
```

CSS Module (`ToolBtn.module.css`):
- `.btn` base: display flex, align-items center, justify-content center, border none, border-radius 6px, padding 0, cursor pointer, color var(--text-secondary), background transparent, transition background-color var(--transition-hover). Also transition color var(--transition-hover).
- `.btn:hover:not(:disabled)` — background-color var(--bg-hover) (for ghost variant; filled variant uses inline style for custom hover color via CSS custom property override)
- `.btn:disabled` — color var(--text-disabled), opacity 0.5, cursor default
- `.btn.active` — for active state (e.g., ToolStrip active button uses --accent-blue bg)
- `.default` — width and height 36px (--size-toolstrip-btn)
- `.small` — width and height 26px (--height-widget-btn)

For the filled variant hover, use a CSS custom property approach: the component sets `--_btn-hover-bg` as an inline style variable, and the CSS uses `.filled:hover:not(:disabled) { background-color: var(--_btn-hover-bg) }`. This avoids onMouseEnter/onMouseLeave while supporting per-instance hover colors.

The component should compose className from CSS module classes using template literals or array join. No external classnames library needed.

**Separator** — Per user decision: two variants (line and gap). Orientation prop for horizontal/vertical.

Props interface:
```typescript
interface SeparatorProps {
  variant?: 'line' | 'gap';          // default: 'line'
  orientation?: 'vertical' | 'horizontal';  // default: 'vertical'
  level?: 'default' | 'subtle' | 'strong';  // border color hierarchy level, default: 'subtle'
  size?: number;                       // length in px (default: 16 for vertical, full-width for horizontal)
  gap?: number;                        // spacing in px for gap variant (default: 8)
}
```

CSS Module (`Separator.module.css`):
- `.line` — flex-shrink 0
- `.line.vertical` — width 1px, margin 0 6px
- `.line.horizontal` — height 1px, width 100%
- Background color controlled by level: `.default` uses var(--border-default), `.subtle` uses var(--border-subtle), `.strong` uses var(--border-strong)
- `.gap` — invisible spacer
- `.gap.vertical` — margin set by gap prop inline
- `.gap.horizontal` — margin set by gap prop inline

Height for vertical line set via inline style from `size` prop (default 16px).
