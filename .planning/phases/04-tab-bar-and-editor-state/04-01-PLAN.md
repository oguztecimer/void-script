<task type="auto">
  <name>Task 2: EditorState map for per-tab state preservation</name>
  <files>editor-ui/src/components/Editor.tsx</files>
  <action>
Refactor Editor.tsx to preserve EditorState (undo history, scroll position) across tab switches. The current implementation destroys and recreates EditorState on every switch, losing all undo history and scroll position.

**Implementation approach — EditorState cache with scroll snapshot:**

1. Add a module-level `Map<string, { state: EditorState; scrollSnapshot: Extension | null }>` called `editorStates` outside the component. This map is keyed by `scriptId` and stores the last EditorState + scroll snapshot when the user switches away from a tab.

2. In the `useEffect` that runs when `activeTabId` changes:
   - BEFORE destroying the old view: save the current EditorView's state and scroll snapshot:
     ```typescript
     if (viewRef.current) {
       const prevId = /* previous activeTabId — track via a ref */;
       if (prevId) {
         editorStates.set(prevId, {
           state: viewRef.current.state,
           scrollSnapshot: viewRef.current.scrollSnapshot(),
         });
       }
       viewRef.current.destroy();
       viewRef.current = null;
     }
     ```
   - Use a `useRef<string | null>` called `prevTabIdRef` to track the previous active tab ID. Update it at the end of the effect.

3. When creating the new EditorView for the newly active tab:
   - Check if `editorStates` has a cached entry for `activeTab.scriptId`
   - If YES: use the cached `state` directly (preserves undo history, selection, etc.). Include the cached `scrollSnapshot` in the extensions. Important: also check if the document content has changed externally (compare `cached.state.doc.toString()` with `activeTab.content`) — if different, create fresh state (content was changed by IPC while tab was inactive).
   - If NO: create a fresh `EditorState.create()` as currently done (first time opening this tab)

4. When closing a tab via `closeTab`, also remove its entry from `editorStates`:
   - Add a `useEffect` that listens to `tabs` array length changes and cleans up orphaned entries:
     ```typescript
     const tabIds = new Set(tabs.map(t => t.scriptId));
     for (const key of editorStates.keys()) {
       if (!tabIds.has(key)) editorStates.delete(key);
     }
     ```

5. The existing `handleUpdate` callback, breakpoint gutter, debug line field, linter, and keymap extensions should all be included in both the fresh and restored paths. For the restored path, the extensions are already baked into the saved EditorState — do NOT re-add them. The `scrollSnapshot` is the only extra extension to add.

6. Keep the current pattern of using `viewRef` for the active EditorView. The map only stores states for INACTIVE tabs.

**Key detail — scroll restoration:** `EditorView.scrollSnapshot()` returns an `Extension` that, when included in the extensions array of a new EditorView, restores the scroll position. Include it when creating the view from a cached state.

**What NOT to do:**
- Do NOT store EditorState in Zustand — EditorState is a large, immutable, non-serializable object. A module-level Map is the correct pattern per CodeMirror docs.
- Do NOT try to keep all EditorViews mounted and hidden — CodeMirror views are heavy DOM consumers. One view at a time, cache the state.
- Do NOT skip the content-change check — if IPC updates the tab content while it's inactive, the cached state would have stale content.
