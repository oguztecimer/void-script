**ToolStrip.tsx changes:**
1. Change the `ToolStripItem` interface `icon` field from `string` to `React.ReactNode`.
2. Pass a `className={isActive ? styles.activeBtn : styles.stripBtn}` prop to `ToolBtn`. Both classes override ToolBtn's default color behavior for strip context.
3. The `ToolBtn` children should be `{item.icon}` (now a ReactNode SVG element instead of text).

**ToolStrip.module.css changes:**
Add these classes:

```css
/* Override ToolBtn's default --text-secondary for strip context */
.stripBtn {
  color: var(--text-tertiary);
}
.stripBtn:hover:not(:disabled) {
  color: var(--text-primary);
}

/* Active strip button: edge indicator instead of solid blue fill */
.activeBtn {
  color: var(--text-primary);
}

/* Left strip: active indicator on RIGHT edge (inner edge) */
.left .activeBtn {
  box-shadow: inset -2px 0 0 var(--accent-blue);
}

/* Right strip: active indicator on LEFT edge (inner edge) */
.right .activeBtn {
  box-shadow: inset 2px 0 0 var(--accent-blue);
}
```

CRITICAL: The `.activeBtn` class is passed via `className` prop to ToolBtn, NOT via the `active` prop. Do NOT pass `active={true}` to ToolBtn for strip buttons — that triggers the solid blue fill from ToolBtn.module.css. Instead, pass `active={false}` (or omit it) and use the className-based edge indicator.
