**Create editor-ui/src/components/BottomTabStrip.tsx:**

A standalone component that replaces the inline `BottomTab` function and the bottom panel header JSX from App.tsx. Wired to Zustand for tab state.

```tsx
import { useStore } from '../state/store';
import { ToolBtn } from '../primitives/ToolBtn';
import styles from './BottomTabStrip.module.css';

const BOTTOM_TABS = [
  { id: 'console', label: 'Console' },
];

export function BottomTabStrip() {
  const activeTab = useStore((s) => s.bottomPanelTab);
  const setTab = useStore((s) => s.setBottomPanelTab);
  const clearConsole = useStore((s) => s.clearConsole);
  const toggleBottomPanel = useStore((s) => s.toggleBottomPanel);

  return (
    <div className={styles.strip}>
      <div className={styles.tabs}>
        {BOTTOM_TABS.map((tab) => (
          <button
            key={tab.id}
            className={`${styles.tab} ${activeTab === tab.id ? styles.active : ''}`}
            onClick={() => setTab(tab.id)}
          >
            {tab.label}
          </button>
        ))}
      </div>
      <div className={styles.actions}>
        <ToolBtn size="small" onClick={clearConsole} title="Clear Console">
          <svg width="12" height="12" viewBox="0 0 16 16">
            <path d="M2 2h12M4 6h8M6 10h4M8 14" stroke="currentColor" strokeWidth="1.5"/>
          </svg>
        </ToolBtn>
        <ToolBtn size="small" onClick={toggleBottomPanel} title="Close">
          <svg width="12" height="12" viewBox="0 0 16 16">
            <path d="M4 4l8 8M12 4l-8 8" stroke="currentColor" strokeWidth="1.5"/>
          </svg>
        </ToolBtn>
      </div>
    </div>
  );
}
```

**Update App.tsx layout:**
Import `{ PanelGroup, Panel, PanelResizeHandle }` from `'react-resizable-panels'`.

Restructure the `.main` div content so that:
1. Left ToolStrip stays OUTSIDE the PanelGroup (fixed width, not resizable)
2. Right ToolStrip stays OUTSIDE the PanelGroup (fixed width, not resizable)
3. Everything between the two tool strips is wrapped in a horizontal PanelGroup
4. Left panel (ScriptList) is a collapsible Panel
5. Center content is the main Panel (tabs + editor + bottom panel)
6. Right panel (DebugPanel) is a collapsible Panel
7. PanelResizeHandle components sit between adjacent Panels

Layout structure:
```tsx
<div className={styles.main}>
  {/* Left tool strip — fixed, OUTSIDE PanelGroup */}
  <ToolStrip side="left" ... />

  {/* Resizable panel layout */}
  <PanelGroup direction="horizontal" autoSaveId="void-main-layout" className={styles.panelGroup}>
    {/* Left panel — always rendered, content conditional */}
    {leftPanelOpen && (
      <>
        <Panel
          id="left-panel"
          order={1}
          defaultSize={18}
          minSize={10}
          maxSize={40}
          collapsible
        >
          <ScriptList />
        </Panel>
        <PanelResizeHandle className={styles.resizeHandle} />
      </>
    )}

    {/* Center panel */}
    <Panel id="center" order={2}>
      <div className={styles.center}>
        <TabBar />
        <div className={styles.editorArea}>
          <Editor />
        </div>
        {bottomPanelOpen && (
          <div className={styles.bottomPanel}>
            <BottomTabStrip />
            <Console />
          </div>
        )}
      </div>
    </Panel>

    {/* Right panel — always rendered when debugging, content conditional */}
    {rightPanelOpen && isDebugging && (
      <>
        <PanelResizeHandle className={styles.resizeHandle} />
        <Panel
          id="right-panel"
          order={3}
          defaultSize={18}
          minSize={10}
          maxSize={40}
          collapsible
        >
          <DebugPanel />
        </Panel>
      </>
    )}
  </PanelGroup>

  {/* Right tool strip — fixed, OUTSIDE PanelGroup */}
  <ToolStrip side="right" ... />
</div>
```

IMPORTANT NOTES:
- `autoSaveId="void-main-layout"` enables localStorage persistence for Phase 7. This is free from react-resizable-panels.
- Side panels are conditionally rendered WITH their PanelResizeHandle. When `leftPanelOpen` is false, neither the Panel nor the handle renders. This is acceptable for Phase 5 DOM prep — Phase 7 will switch to imperative `panel.collapse()`/`expand()` for smoother behavior.
- `order` props ensure correct layout order even when panels mount/unmount.
- The center Panel has no `defaultSize` — it takes remaining space automatically.
- `minSize={10}` and `maxSize={40}` are percentages, roughly corresponding to the locked constraint "min 150px, max 50% viewport" on a typical ~1400px window. Phase 7 will refine these with pixel-to-percentage conversion.
