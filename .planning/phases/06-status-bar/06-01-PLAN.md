**2. Create `editor-ui/src/components/NavPath.tsx`:**

NavPath derives path segments from the active tab and renders clickable breadcrumbs.

Path derivation logic (pure function, no new state needed):
- Always start with project segment: `{ label: 'VOID//SCRIPT', kind: 'project' }`
- If no active tab: return only the project segment
- If active tab exists: find matching `ScriptInfo` in `scriptList` by `scriptId`
- If `info.script_type` exists and has a label in `TYPE_LABELS`: insert a folder segment `{ label: TYPE_LABELS[info.script_type], kind: 'folder' }` between project and file
- File segment: `{ label: activeTab.name, kind: 'file' }`
- Flat case result: `VOID//SCRIPT › miner_brain.vs`
- Typed case result: `VOID//SCRIPT › Ship Brains › miner_brain.vs`

Rendering rules:
- Each segment except the last renders as a `<button>` with the chevron ` › ` character appended INSIDE the button (larger click target per CONTEXT.md)
- The chevron character is wrapped in a `<span className={styles.chevron}>` inside the button so it stays `--text-secondary` even on hover
- Last segment (file) renders as a `<button>` with NO trailing chevron
- Project segment always renders (even when no file open)

Click handlers:
- Project name click: call `toggleLeftPanel()` (always, regardless of whether a file is open)
- Folder segment: render as `<span>` (NOT `<button>`) — no onClick, no `cursor: pointer`. Folder filtering is deferred per CONTEXT.md
- File segment: no-op for now (scroll-to-top can be added later). Render as `<span>` not `<button>` to avoid implying functionality

Selector usage — use these Zustand selectors:
```typescript
const activeTabId = useStore((s) => s.activeTabId);
const tabs = useStore((s) => s.tabs);
const scriptList = useStore((s) => s.scriptList);
const toggleLeftPanel = useStore((s) => s.toggleLeftPanel);
const activeTab = tabs.find((t) => t.scriptId === activeTabId);
```

Import `TYPE_LABELS` from `'../state/scriptTypes'`.
