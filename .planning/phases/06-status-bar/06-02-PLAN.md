  <action>
Add `flex-shrink: 0;` to the `.bar` rule in `editor-ui/src/components/StatusBar.module.css`.

The `.bar` element is a direct child of `.app` (a `flex-direction: column` container with `height: 100vh`). The sibling `.main` div has `flex: 1` which makes it greedy — without `flex-shrink: 0`, the StatusBar's `height: var(--height-statusbar)` (24px) is overridden by the default `flex-shrink: 1`, compressing it to 0px.

The fix is a single property addition. The resulting `.bar` rule should be:

```css
.bar {
  display: flex;
  align-items: center;
  height: var(--height-statusbar);
  flex-shrink: 0;
  background-color: var(--bg-panel);
  font-size: var(--font-size-status);
  color: var(--text-secondary);
  border-top: 1px solid var(--border-strong);
  gap: 0;
}
```

Place `flex-shrink: 0` immediately after `height` for logical grouping (sizing properties together).

Do NOT modify any other file. Do NOT change the .spacer rule.
