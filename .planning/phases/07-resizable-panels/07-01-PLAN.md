**Add double-click handlers for all three Separators:**

1. Create `useCallback` handlers for each separator. Use `e.preventDefault()` on all to prevent text selection on double-click (per research Open Question 3):

```tsx
  e.preventDefault();

  if (!panel) return;

    useStore.getState().toggleLeftPanel();
  } else {
    panel.collapse();
    useStore.getState().toggleLeftPanel();
  }
}, []);

const handleRightSeparatorDoubleClick = useCallback((e: React.MouseEvent) => {
  e.preventDefault();
  const panel = rightPanelRef.current;
  if (!panel) return;
  if (panel.isCollapsed()) {
    panel.expand();
    useStore.getState().toggleRightPanel();
  } else {
    panel.collapse();
    useStore.getState().toggleRightPanel();
  }
}, []);

const handleBottomSeparatorDoubleClick = useCallback((e: React.MouseEvent) => {
  e.preventDefault();
  const panel = bottomPanelRef.current;
  if (!panel) return;
  if (panel.isCollapsed()) {
    panel.expand();
    setBottomPanelOpen(true);
  } else {
    panel.collapse();
    setBottomPanelOpen(false);

}, [setBottomPanelOpen]);
```

NOTE on left/right handlers: Since `toggleLeftPanel` and `toggleRightPanel` flip the boolean, and the existing useEffect watches the state to call expand/collapse, it is simpler to just call the toggle function. However the double-click handler should call the imperative API directly AND sync state to avoid the animation delay from the useEffect round-trip. The pattern: call `panel.collapse()` / `panel.expand()` directly in the handler, then update Zustand. The useEffect will see the state already matches and be a no-op.

Refined approach for left/right — avoid `toggle` since we know the current state from `panel.isCollapsed()`:

For left:
```tsx
const handleLeftSeparatorDoubleClick = useCallback((e: React.MouseEvent) => {
  e.preventDefault();
  const panel = leftPanelRef.current;
  if (!panel) return;
  // toggleLeftPanel flips the boolean; useEffect will respond
  // but we also call imperative API for immediate visual feedback
  toggleLeftPanel();
}, [toggleLeftPanel]);
```

Actually, the simpler correct approach: just call `toggleLeftPanel()` — the existing useEffect already calls `panel.expand()` or `panel.collapse()`. The 150ms animation class handles the transition. This is the cleanest pattern and avoids duplicating imperative calls.

3. Add `onResize` to the bottom Panel for drag-collapse Zustand sync (per research Pattern 4):

```tsx
<Panel
  panelRef={bottomPanelRef}
  id="bottom-panel"
  defaultSize="25%"
  minSize="10%"
  maxSize="50%"
  collapsible
  collapsedSize={0}
  className={!isResizing ? styles.panelAnimated : ''}
  onResize={(size) => {
    // Sync Zustand when panel collapses/expands via drag interaction
    // Use < 1 threshold instead of === 0 for floating point safety (research Open Question 1)
    const isNowCollapsed = size.asPercentage < 1;
    if (isNowCollapsed && bottomPanelOpen) {
      setBottomPanelOpen(false);
    } else if (!isNowCollapsed && !bottomPanelOpen) {
      setBottomPanelOpen(true);
    }
  }}
>
```

This guard (`isNowCollapsed && bottomPanelOpen` / `!isNowCollapsed && !bottomPanelOpen`) prevents the useEffect feedback loop described in research Pitfall 2 — Zustand is only updated when the state doesn't already match.
