Create `editor-ui/src/primitives/Tooltip.tsx`:
- Interface: `TooltipProps { content: string; children: React.ReactNode; disabled?: boolean; }`
- Wrapper div with `display: inline-flex; position: relative` (NOT `display: contents` -- it breaks absolute positioning).
- On `mouseenter`: start 800ms setTimeout to set `visible: true`. On `mouseleave`: clearTimeout + set `visible: false` instantly.
- When visible, render a `<div>` with class `styles.tooltip`, positioned `absolute` below trigger (`top: calc(100% + 4px); left: 50%; transform: translateX(-50%)`).
- Viewport flip: use a `useEffect` + `useRef` on the tooltip div. When visible becomes true, check `getBoundingClientRect().bottom > window.innerHeight`. If clipped, add a `styles.flipped` class that sets `bottom: calc(100% + 4px); top: auto`.
- Animation: CSS `animation: fadeIn 100ms ease` on mount. No exit animation.
- If `content` is empty string or `disabled` is true, render children only (no wrapper overhead if possible, or just skip the tooltip div).

Create `editor-ui/src/primitives/Tooltip.module.css`:
- `.wrapper`: `position: relative; display: inline-flex;`
- `.tooltip`: `position: absolute; top: calc(100% + 4px); left: 50%; transform: translateX(-50%); background: #3C3F41; color: #BBBBBB; border: 1px solid #555555; border-radius: 4px; padding: 4px 8px; font-size: 12px; font-family: var(--font-ui); white-space: nowrap; pointer-events: none; z-index: 1000;` with `animation: fadeIn 100ms ease;`
- `.flipped`: `top: auto; bottom: calc(100% + 4px);`
- `@keyframes fadeIn { from { opacity: 0 } to { opacity: 1 } }`
- No box-shadow (per locked spec). No arrow/caret.

Modify `editor-ui/src/primitives/ToolBtn.tsx`:
- Add `shortcut?: string` to `ToolBtnProps` interface.
- Import `Tooltip` from `'./Tooltip'`.
- Compute tooltip content: `const tooltipContent = shortcut ? `${title} (${shortcut})` : title;`
- Wrap the `<button>` element with `<Tooltip content={tooltipContent} disabled={disabled}>`.
- REMOVE the native `title={title}` attribute from the `<button>` element entirely (Pitfall 3: native title causes double tooltip).
- Preserve all existing button behavior (className, onClick, disabled, style, children).
