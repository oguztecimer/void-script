In Editor.tsx, after the `new EditorView(...)` call (line 249) and before the scroll restore block, add a setCursor call that reads the real selection head from editorState:

```typescript
viewRef.current = new EditorView({
  state: editorState,
  parent: containerRef.current,
});

// Sync initial cursor position so BreadcrumbBar sees the correct line immediately.
// editorState.selection.main.head is the character offset of the selection head.
{
  const head = editorState.selection.main.head;
  const line = editorState.doc.lineAt(head);
  useStore.getState().setCursor(line.number, head - line.from + 1);
}

// Restore scroll position after view is mounted
if (cached?.scrollSnapshot) {
  viewRef.current.dispatch({ effects: cached.scrollSnapshot });
}
```

Use a block scope `{ }` to avoid polluting the surrounding useEffect scope with local variables `head` and `line`.

This runs on every tab switch because the useEffect depends on `activeTabId`. The handleUpdate listener already calls setCursor on every subsequent cursor movement — this call covers only the moment the view is first mounted.

Do not change any other part of Editor.tsx.
