2. Extend the useEffect block. After computing the vertical flip, add horizontal clamping:

```typescript
useEffect(() => {
  if (visible && tooltipRef.current) {
    const rect = tooltipRef.current.getBoundingClientRect();

    // Vertical flip (existing behavior — preserve exactly)
    if (rect.bottom > window.innerHeight) {
      setFlipped(true);
    } else {
      setFlipped(false);
    }

    // Horizontal clamping — compute corrective delta to keep tooltip in viewport
    const MARGIN = 4; // px gap from viewport edge
    let dx = 0;
    if (rect.left < MARGIN) {
      dx = MARGIN - rect.left;          // shift right
    } else if (rect.right > window.innerWidth - MARGIN) {
      dx = (window.innerWidth - MARGIN) - rect.right;  // shift left (negative)
    }
    setOffsetX(dx);
  }
  if (!visible) {
    setFlipped(false);
    setOffsetX(0);
  }
}, [visible]);
```

3. Apply offsetX as an inline style override on the tooltip div. The CSS already does `transform: translateX(-50%)`. Override with a calculated value that incorporates both the centering offset and the correction:

```tsx
{visible && (
  <div
    ref={tooltipRef}
    className={`${styles.tooltip}${flipped ? ` ${styles.flipped}` : ''}`}
    style={offsetX !== 0 ? { transform: `translateX(calc(-50% + ${offsetX}px))` } : undefined}
  >
    {content}
  </div>
)}
```

Important: `style` is only set when `offsetX !== 0`. When centered in the viewport, `style` is `undefined` and the CSS `transform: translateX(-50%)` applies unmodified. This avoids overriding CSS unnecessarily.

Do not change the timer logic, the flipped CSS class, the wrapper div, or the TooltipProps interface.
